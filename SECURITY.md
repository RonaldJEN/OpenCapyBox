# Security Policy

## Supported Versions

目前支持安全更新的版本：

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

> 注：本项目仍处于早期开发阶段，建议始终使用最新版本。

## Reporting a Vulnerability

如果您发现安全漏洞，请**不要**在公开的 Issue 中报告。

### 报告方式

1. **GitHub Security Advisories**：使用 [GitHub 私密漏洞报告](https://github.com/RonaldJEN/OpenCapyBox/security/advisories/new)（推荐）
2. **Email**：发送邮件至 renyx0430@163.com（建议邮件标题前缀使用 `[Security]`）

### 报告内容

请在报告中包含：

- 漏洞描述
- 复现步骤
- 受影响的版本
- 可能的影响范围
- 如有可能，提供修复建议

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 天内
- **修复发布**：根据严重程度，14-90 天

### 处理流程

1. 我们会确认收到您的报告
2. 评估漏洞严重程度
3. 开发修复方案
4. 发布安全更新
5. 公开致谢（如您同意）

感謝您幫助我們保持項目安全！
